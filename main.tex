\begin{abstract}
Instrumented Timed Up and Go (TUG) analysis can support clinical and research decision-making, but robust and reproducible markerless pipelines are still limited. We present \textit{tugturn}, a Python-based workflow for 3D markerless TUG processing that combines phase segmentation, gait-event detection, spatiotemporal metrics, intersegmental coordination, and dynamic stability analysis. The pipeline uses spatial thresholds to segment each trial into stand, first gait, turning, second gait, and sit phases, and applies a relative-distance strategy to detect heel-strike and toe-off events within valid gait windows. In addition to conventional kinematics, \textit{tugturn} provides Vector Coding outputs and Extrapolated Center of Mass (XCoM)-based metrics. The software is configured through TOML files and produces reproducible artifacts, including HTML reports, CSV tables, and quality-assurance visual outputs. A complete runnable example is provided with test data and command-line instructions. This manuscript describes the implementation, outputs, and reproducibility workflow of \textit{tugturn} as a focused software contribution for markerless biomechanical TUG analysis.
\end{abstract}

\section{Introduction}
Timed Up and Go (TUG) is widely used to evaluate mobility and functional performance in clinical populations because it integrates postural transition, walking, turning, and sitting in a single task \citep{podsiadlo1991}. Traditional motion-capture workflows provide high-fidelity biomechanical information, but they often depend on specialized infrastructure and time-consuming processing pipelines. Markerless approaches based on computer vision have expanded access to movement analysis and enabled broader deployment in rehabilitation and applied biomechanics \citep{lugaresi2019mediapipe, nath2019using}.

Historically, gait events and turning metrics have been analyzed predominantly using standard optoelectronic kinematic systems, instrumented walkways, or force platforms \citep{khobkhun2022investigation, thanakamchokchai2025effects}. However, widely available full-body 3D-landmark frameworks custom-built for automated clinical phase segmentation remain scarce \citep{tahara2025predicting}. Without such specialized automated markerless tools, analyses in clinical or uncontrolled settings often resort to subjective visual recognition or labor-intensive manual frame-by-frame data processing. To overcome these limitations, the \textit{tugturn} method was developed to automate these demanding procedures, bridging the gap between raw video tracking and robust biomechanical endpoints.

Despite this progress, many markerless pipelines still emphasize either pose extraction or generic gait summaries and do not provide a structured TUG-specific workflow with robust phase-aware event handling. In TUG, meaningful interpretation depends on correctly isolating movement phases (e.g., first gait versus second gait) and on reducing false event detections outside valid locomotor windows. This issue is particularly relevant in heterogeneous clinical recordings, where pauses, partial turns, and variable trajectories are common.

The \textit{tugturn} module was designed to address this gap by integrating: (i) spatially constrained phase segmentation, (ii) gait-event detection based on relative-distance principles \citep{zeni2008}, (iii) phase-specific spatiotemporal metrics, (iv) Vector Coding for coordination analysis, and (v) Extrapolated Center of Mass (XCoM) indicators for dynamic stability assessment. The goal of this paper is to document the software architecture and reproducible processing flow of \textit{tugturn} as a practical tool for markerless 3D TUG analysis.

\section{Methods and Implementation}
\subsection{Input Data and Processing Model}
\textit{tugturn} processes MediaPipe-based 3D CSV files generated from markerless pose estimation pipelines \citep{lugaresi2019mediapipe}. Execution supports both single-file and batch-directory modes through a command-line interface. Subject and trial parameters can be supplied in TOML files, allowing standardized runs across datasets.

The core implementation is organized around a \texttt{TUGAnalyzer} class in \texttt{tugturn.py}, which computes kinematic variables, segment boundaries, gait events, and report-ready outputs from a unified data structure.

To demonstrate its applicability in real-world clinical scenarios, the \textit{tugturn} algorithm was evaluated on a comprehensive dataset comprising \textbf{324 continuous markerless kinematic sequences}. These data were collected from \textbf{37 individuals} with mild-to-moderate Parkinson's disease. The experimental protocol spanned three distinct assessment moments (baseline, immediately post-intervention, and a one-week follow-up), consisting of up to three TUG trials per moment: one requiring a right turn, one a left turn, and one allowing a free-choice turn. Out of 333 total acquired trials, the automated pipeline successfully processed 324 sequences, yielding a 97.3\% retention rate. The minor data attrition (9 trials) resulted strictly from external factors such as camera occlusions or upstream tracking artifacts. This high processing success rate underscores the robustness of the pipeline when handling the natural variability of clinical recordings.

For the reference run described in this manuscript, the 3D input file is
\texttt{tugturn/s26\_m1\_t1.csv}. This file contains the frame-wise markerless
3D trajectories (MediaPipe-derived landmarks) used to compute CoM, kinematics,
gait events, and phase segmentation.

\subsection{Phase Segmentation}

Gait-event detection and turn segmentation in the TUG were performed in the tugturn.py module using a hybrid spatiotemporal approach. First, a walking mask was constructed to restrict analysis to locomotion periods by combining upright posture (derived from the pelvis vertical trajectory) with horizontal progression velocity. Next, heel-strike (HS) and toe-off (TO) events were extracted by projecting heel and forefoot markers onto the pelvis unit progression vector, with temporal smoothing and minimum peak distance/prominence constraints to reduce false positives. 
The turn phase was defined using anteroposterior spatial rules (chair threshold and turn zone) and refined through axial kinematics, using the peak rate of change in the shoulder yaw angle to identify turn180 onset and offset. Finally, HS/TO events outside first and second gait windows were removed, ensuring biomechanical consistency between event detection and functional TUG phase segmentation.

Each trial is segmented into:
\begin{itemize}
  \item \texttt{stand}
  \item \texttt{gait\_1}
  \item \texttt{turn180}
  \item \texttt{gait\_2}
  \item \texttt{sit}
\end{itemize}

Segmentation combines temporal logic and spatial thresholds along the anteroposterior axis. To ensure adaptability across distinct experimental setups and room dimensions, the fundamental limits of the capture area are fully configurable by the user via command-line flags or the TOML configuration file:
\begin{itemize}
  \item \texttt{-y/y\_chair}: Defines the AP coordinate distinguishing the chair from the walking path.
  \item \texttt{-t/y\_turn}: Defines the AP coordinate indicating the target turn zone.
  \item \texttt{y\_chair\_tol}: Specifies the spatial tolerance around these anchors.
\end{itemize}
By explicitly exposing these flags, researchers can align the physical constraints of the automated pipeline with their specific recorded protocol without modifying the underlying source code. The implementation uses smoothed CoM and foot trajectories to enforce clinically meaningful boundaries:
\begin{enumerate}
  \item \textbf{Vertical transition detection (CoM\_z):} \texttt{stand} and \texttt{sit} boundaries are initialized from percentile-based CoM\_z amplitude.
  \item \textbf{First gait start:} first frame where a foot crosses the chair threshold (\texttt{y\_chair}).
  \item \textbf{Turn-zone anchoring:} local maximum progression along Y near \texttt{y\_turn} (with tolerance).
  \item \textbf{Pause and turn split:} low trunk-speed anchor followed by shoulder-yaw-rate peak to delimit \texttt{stop\_5s} and \texttt{turn180}.
  \item \textbf{Second gait end:} foot crossing back below \texttt{y\_chair}; remaining frames define \texttt{sit}.
\end{enumerate}

This hybrid spatial-temporal design improves robustness for slow, hesitant, and clinically irregular trajectories.

\subsection{Gait Events and Kinematic Metrics}
Heel-strike (HS) and toe-off (TO) events are estimated using a relative-distance strategy inspired by \citet{zeni2008}, with phase-aware masking for TUG. The method computes pelvis progression direction in the horizontal plane and projects heel/toe vectors onto this direction:
\[
p_{\text{heel}}(t)=\langle \mathbf{heel}_{xy}(t)-\mathbf{pelvis}_{xy}(t), \hat{\mathbf{d}}_{xy}(t)\rangle
\]
\[
p_{\text{toe}}(t)=\langle \mathbf{toe}_{xy}(t)-\mathbf{pelvis}_{xy}(t), \hat{\mathbf{d}}_{xy}(t)\rangle
\]
HS events are detected as peaks of $p_{\text{heel}}(t)$ and TO events as peaks of $-p_{\text{toe}}(t)$, after smoothing and minimum inter-event distance constraints.

A walking mask (standing posture + pelvis macro-velocity threshold) is used to suppress false events during sit-to-stand, pause, turning, and stand-to-sit. After phase segmentation, events outside \texttt{gait\_1} and \texttt{gait\_2} are removed.

Compared to traditional gait analysis—where steps are extracted via clean trajectories from reflective markers or force plate impulses—markerless tracking relies heavily on pixel-based machine learning inference, which often introduces high-frequency noise and keypoint flickering \citep{tahara2025predicting}. To mitigate this instability, particularly at the distal extremities, \textit{tugturn} computes the spatial average of multiple adjacent foot landmarks (e.g., ankle, heel, and toe-tip) to represent a more stable unified foot position when necessary. Furthermore, because sagittal limb velocity and acceleration signs can flip erroneously during hesitant gait or tracking glitches, the pipeline aggregates these derivatives into a 2D resultant scalar called Kinematic Magnitude ($|V_{yz}|$ and $|A_{yz}|$). This strategy, alongside the robust fictitious event filtering, ensures that spatiotemporal parameters reflect genuine locomotion cycles even under heterogeneous clinical conditions.

The software computes:
\begin{itemize}
  \item joint angles (e.g., knee and ankle, right/left),
  \item trunk inclination,
  \item trunk and foot kinematics in the sagittal YZ plane (position, velocity, acceleration),
  \item global and phase-specific spatiotemporal parameters.
\end{itemize}
To reduce sign ambiguity in sagittal derivatives, \textit{tugturn} also reports 2D resultant kinematic magnitude:
\[
|V_{yz}| = \sqrt{V_y^2 + V_z^2}, \qquad
|A_{yz}| = \sqrt{A_y^2 + A_z^2}.
\]

\subsection{Coordination and Stability Metrics}
\textbf{Vector Coding.} The pipeline computes coupling-angle series for trunk--pelvis and limb coordination analyses using:
\[
\gamma = \operatorname{atan2}(\Delta \theta_{\text{distal}}, \Delta \theta_{\text{proximal}})\;\;\text{mapped to}\;\;[0,360)
\]
For axial analysis, $\theta$ corresponds to trunk and pelvis yaw (XY plane). The coupling angle is classified into four canonical bins (\textit{In-Phase}, \textit{Anti-Phase}, \textit{Proximal-Phase}, \textit{Distal-Phase}), and summarized as percentage occupancy plus Coupling Angle Variability (CAV, circular SD). A limb Y-axis vector-coding variant is also computed during the first and second gait phases.

\textbf{XCoM.} Dynamic stability is estimated from Extrapolated Center of Mass:
\[
XCoM = CoM + \frac{CoM_{vel}}{\omega_0}, \qquad
\omega_0 = \sqrt{\frac{g}{l}},
\]
where $l$ is approximated by mean vertical CoM. Phase-level XCoM deviations are exported as stability descriptors for the first and second gait.

\subsection{Algorithmic Workflow (Pseudocode)}

\begin{algorithm}[H]
\caption{Hybrid Detection of Gait Events and Turn Phase in TUG}
\textbf{Input:} 3D trajectories of pelvis, heels, toes, shoulders; sampling frequency $f_s$ \\
\textbf{Output:} $\{HS_R, TO_R, HS_L, TO_L\}$ and phase bounds $\{stand, gait\_1, stop, turn180, gait\_2, sit\}$
\begin{algorithmic}[1]
\State Compute pelvis center $\mathbf{p}_{pelvis}(t)$ and smooth signals
\State Estimate standing mask from pelvis vertical trajectory percentiles
\State Compute pelvis horizontal speed $v_{xy}(t)$
\State \textbf{WalkingMask} $\gets$ \textbf{StandingMask} $\land \left(v_{xy}(t) > \tau_v\right)$

\State Estimate unit progression vector $\hat{\mathbf{d}}(t)$ from smoothed pelvis velocity
\For{each side $s \in \{Right, Left\}$}
    \State Project heel-to-pelvis vector onto $\hat{\mathbf{d}}(t)$
    \State Apply \textbf{WalkingMask} and detect local maxima $\rightarrow HS_s$
    \State Project toe-to-pelvis vector onto $\hat{\mathbf{d}}(t)$
    \State Invert projected signal, apply \textbf{WalkingMask}, detect maxima $\rightarrow TO_s$
    \State Enforce minimum inter-event distance and peak prominence
\EndFor

\State Detect first-gait start at first frame where foot AP coordinate crosses $y_{chair}$
\State Detect turn-zone anchor near AP peak ($y_{turn}\pm y_{tol}$)
\State Find stop anchor as minimum smoothed trunk-speed window near turn zone
\State Compute shoulder yaw rate $\omega_{yaw}(t)$ from shoulder vector angle derivative
\State Identify turn peak as $\arg\max(\omega_{yaw})$ in post-stop search window
\State Expand second-gait/first-gait from peak until $\omega_{yaw}(t) < \tau_{yaw}$ to obtain $(turn\_start, turn\_end)$
\State Define second-gait start from sustained negative AP velocity
\State Define second-gait end when feet cross below $y_{chair}$
\State Enforce contiguous non-negative phase durations
\State Remove HS/TO events outside $\{gait\_1, gait\_2\}$
\State \Return gait events and TUG phase segmentation
\end{algorithmic}
\end{algorithm}

\section{Discussion}
\textit{tugturn} contributes a targeted software layer for TUG-specific markerless analysis, emphasizing phase-aware processing rather than generic full-trial summaries. This focus is important for protocols where turning, sit-to-stand transitions, and return gait carry distinct clinical meaning.

From a software-engineering perspective, the combination of CLI execution, TOML configuration, and structured outputs facilitates reproducibility, batch scaling, and downstream statistical workflows. The report stack (HTML plus CSV) supports both exploratory interpretation and scripted analysis.

Current limitations include dependence on upstream pose-estimation quality, protocol-specific threshold tuning, and the absence of benchmarked agreement against instrumented laboratory gold standards in this manuscript. Future work should include cross-dataset validation, uncertainty quantification, and multicenter reproducibility studies.

\section{Conclusion}
This work presents \textit{tugturn} as a reproducible and practical software pipeline for markerless 3D TUG analysis. By combining spatial phase segmentation, phase-filtered gait events, Vector Coding, and XCoM metrics in a single workflow, \textit{tugturn} enables consistent extraction of clinically relevant biomechanical descriptors from routine video-derived data. The provided test workflow and command-line interface support transparent reuse and adaptation in biomechanics and rehabilitation research.
